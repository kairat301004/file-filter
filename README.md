# file-filter
Программа написана на Java 21. Сборка через Maven 3.9.10. Сторонние библиотеки не используются.

Я заранее подготовил и добавил 2 текстовых файла - in1.txt и in2.txt в проект. 
Они содерджат необходимые данные для работы в программе)

## Как собрать

Сначала клонируйте репозиторий и перейдите в папку проекта:
```bash
git clone https://github.com/kairat301004/file-filter
cd file-filter
```


Соберите jar-файл командой:
```bash
mvn clean package
```

После сборки jar-файл появится в папке target.

## Как запустить
Без опций
```bash
java -jar target/file-filter-1.0-SNAPSHOT.jar in1.txt in2.txt
```

Результаты появятся в текущей папке в файлах integers.txt, floats.txt, strings.txt

Статистика не выводится

Пример с статистикой и префиксом
```bash
java -jar target/file-filter-1.0-SNAPSHOT.jar -s -p sample- in1.txt in2.txt
```

Файлы будут иметь префикс sample-
В консоли вы увидите количество элементов каждого типа

## Далее можно применить другие опции.
Вы можете задать папку для выходных файлов через -o, префикс имен через -p и включить режим добавления к существующим файлам с помощью -a. 
Программа умеет собирать статистику: -s выводит краткую статистику (количество элементов), -f  полную (для чисел минимум, максимум, сумма и среднее, для строк - длина самой короткой и длинной строки).

Программа устойчива к ошибкам: ошибки чтения/записи выводятся в консоль, программа продолжает работу

Файлы создаются только если есть данные соответствующего типа

Код разделён на классы: Main, Config, FileProcessor, Statistics, DataType
